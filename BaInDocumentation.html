1. Introduction

1.1 Purpose

The purpose of this document is to present a detailed description of the web application BaIn (Baby Info Manager). It will explain the purpose and features of the software, the interfaces of the software, what the software will do and the constraints under which it must operate. This document is intended for the potential users of the software.

1.2 Intended Audience and Reading Suggestions

The users of the applications are mainly represented by families.

1.3 Product Scope

The purpose of the application is to develop a platform for the management of important information about kids, like feeding and sleeping schedule, medical history and to store media resources (photos, videos).

2. Overall Description

2.1 Product Functions

  • The users will have access to the following features:
    • Authentication:
      • Register
      • Login
    • Add/delete children and information about them.
    • Export medical history in CSV format.
    • See shelters/rescue routes.

2.2 Operating Environment

The developed product can be used on computer devices with a stable connection to the Internet and an installed browser.

2.3 Design and Implementation Constraints

Frontend

  1. HTML
  2. CSS
  3. JavaScript

Backend

  1. Node.js
  2. JSON
  3. MySQL Database

2.4 User Documentation

A user manual will be created.

2.6 Assumptions and Dependencies

  • Jsonwebtoken - JWT provider
  • Bcrypt - Hashing password
